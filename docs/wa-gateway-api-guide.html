Ringkasan cepat

Prasyarat

  • Node.js 20+ dan npm terpasang.
  • WhatsApp yang aktif untuk dipasangkan via QR.
  • Server wa-gateway berjalan (default PORT=5001).

Jalankan singkat

npm install
PORT=5001 KEY=<master-key-opsional> npm start
# Buka QR: http://localhost:5001/session/start?session=mysession

Alur inti

  • Buat device + token → scan QR.
  • Kirim pesan via endpoint V1 (form) atau V2 (JSON).
  • Opsional: set webhook untuk event masuk.

Autentikasi & key

Token mewakili satu device/session. Secret key pada format lama diabaikan.

MetodeContohCatatan
Header AuthorizationAuthorization: 4fd23...Bisa juga Bearer <token> atau <token>.<secret> (secret diabaikan).
Query?token=4fd23...Dipakai bila tidak ingin mengirim header.
Master keykey: <KEY>Wajib jika env KEY di-set; bisa via header atau query ?key=....

Mapping: token → sessionId melalui wa_credentials/device-registry.json. Semua aksi device & pengiriman pesan memilih session berdasar token.

Device & token lifecycle

POST /api/device/create

Membuat device + session dan mengembalikan token serta QR (jika perlu).

curl -X POST http://localhost:5001/api/device/create \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "name=My Device&phone=62812xxxx"
{
  "status": true,
  "message": "create device successfully",
  "data": {
    "device_name": "My Device",
    "device": "62812xxxx",
    "token": "generated-token",
    "qr": "qr-string-or-null"
  }
}
GET /api/device/scan

Memulai session dan mengembalikan QR. Gunakan token pada header/query.

GET /api/device/info

Lihat status device.

{
  "status": "success",
  "message": "Success",
  "data": [
    { "phone": "62812...@s.whatsapp.net",
      "device": "sessionId",
      "status": "online",
      "webhook_url": "https://..." }
  ]
}
POST /api/device/disconnect

Logout session tanpa menghapus registry.

POST /api/device/delete

Logout + hapus registry device dari wa_credentials/device-registry.json.

Endpoint V1 (form-data / query)

Gunakan saat Anda butuh kompatibilitas penuh dengan format lama (form-urlencoded atau query). Semua endpoint memilih session berdasar token.

Kiriman teks

GET /api/send-message
curl "http://localhost:5001/api/send-message?token=TOKEN&phone=62812xxx&message=halo&isGroup=false"
POST /api/send-message
curl -X POST http://localhost:5001/api/send-message \
  -H "Authorization: TOKEN" \
  -d "phone=62812xxx" \
  -d "message=halo" \
  -d "isGroup=false" \
  -d "ref_id=opsional"
{"status":true,"message":"Success","data":{"phone":"...","message":"...","ref_id":null,"id":"..."}}

Media

POST /api/send-image
  • Fields: phone, (image atau url), caption opsional.
  • isGroup opsional.
POST /api/send-video
  • Fields: phone, (video atau url), caption/message opsional.
  • isGroup opsional.
POST /api/send-document
  • Fields: phone, (document atau url), filename opsional (default document.pdf).
  • caption/message opsional, isGroup opsional.
POST /api/send-audio
  • Fields: phone, (audio atau url), ptt/asVoiceNote opsional.
  • isGroup opsional.

Registry Sync (token/apiKey)

Endpoint ini menyediakan isi wa_credentials/device-registry.json agar aplikasi eksternal bisa sinkron token secara otomatis.

GET /admin/device-registry
  • Respons: array JSON berisi token, apiKey, sessionId, dll.
  • Direkomendasikan pakai autentikasi.

Autentikasi

Atur salah satu di .env wa-gateway:

REGISTRY_TOKEN=token_rahasia

atau Basic Auth:

REGISTRY_USER=admin
REGISTRY_PASS=secret

Contoh request

curl -H "Authorization: Bearer token_rahasia" \
  http://localhost:5001/admin/device-registry
curl -u admin:secret \
  http://localhost:5001/admin/device-registry

Jika tidak di-set, endpoint terbuka (tidak disarankan untuk server publik).

Ekspor token ke .env

Jika aplikasi lain membaca token lewat env (mis. WA_GATEWAY_TOKEN), ambil dari wa_credentials/device-registry.json dan tulis ke .env. Pilih skenario:

Satu VPS, beda folder

Contoh: wa-gateway di /var/www/wa, app lain di /var/www/beta/.env.

#!/usr/bin/env bash
set -euo pipefail
WA_DIR="/var/www/wa"
BETA_ENV="/var/www/beta/.env"
SESSION_ID="" # isi jika mau session tertentu
REG="$WA_DIR/wa_credentials/device-registry.json"
token=$(node - <<'NODE'
const fs=require('fs');
const path=process.env.REG_PATH;
const sid=process.env.SESSION_ID||"";
const list=JSON.parse(fs.readFileSync(path,'utf8'));
const pick=sid?(list.find(r=>r.sessionId===sid)||{}):(Array.isArray(list)?list[0]||{}:{});
const t=pick.token||pick.apiKey||"";
if(!t) throw new Error("Token not found");
console.log(t);
NODE
REG_PATH="$REG" SESSION_ID="$SESSION_ID")
tmp=$(mktemp)
grep -v '^WA_GATEWAY_TOKEN=' "$BETA_ENV" 2>/dev/null || true >"$tmp"
echo "WA_GATEWAY_TOKEN=$token" >>"$tmp"
mv "$tmp" "$BETA_ENV"
echo "Updated $BETA_ENV with WA_GATEWAY_TOKEN=$token"

Jalankan: simpan sebagai sync-token-env.sh, chmod +x sync-token-env.sh, lalu ./sync-token-env.sh. Sesuaikan path/SESSION_ID.

Contoh kasus: setelah update apiKey session 6285158663803, jalankan script agar .env app lain ikut terisi token baru.

Dua VPS (wa.watumalang.online → beta.watumalang.online)

Ambil token via SSH dari host wa-gateway, tulis ke .env di host lain.

#!/usr/bin/env bash
set -euo pipefail
WA_HOST="deploy@wa.watumalang.online"
BETA_HOST="deploy@beta.watumalang.online"
WA_DIR="/var/www/wa"
BETA_ENV="/var/www/beta/.env"
SESSION_ID=""
token=$(ssh "$WA_HOST" "REG='$WA_DIR/wa_credentials/device-registry.json'; node -e \"
const fs=require('fs');
const list=JSON.parse(fs.readFileSync(process.env.REG,'utf8'));
const sid='${SESSION_ID}';
const pick=sid?(list.find(r=>r.sessionId===sid)||{}):(Array.isArray(list)?list[0]||{}:{});
const t=pick.token||pick.apiKey||'';
if(!t) throw new Error('Token not found');
console.log(t);
\"")
ssh "$BETA_HOST" "tmp=\$(mktemp); \
  grep -v '^WA_GATEWAY_TOKEN=' '$BETA_ENV' 2>/dev/null || true >\$tmp; \
  echo 'WA_GATEWAY_TOKEN=$token' >>\$tmp; \
  mv \$tmp '$BETA_ENV'; \
  echo 'Updated $BETA_ENV'"

Jalankan: simpan sebagai sync-token-remote.sh, chmod +x, lalu ./sync-token-remote.sh dari mesin yang bisa SSH ke kedua host. Sesuaikan host, path, dan SESSION_ID.

Webhook (opsional)

Set WEBHOOK_BASE_URL di .env untuk menerima event real-time.

  • Session event: POST /webhook/session
  • Message event: POST /webhook/message

Pastikan endpoint Anda idempotent dan mampu menerima update status online/offline serta pesan masuk.

Respons & keterbatasan

  • Respons umum: { "status": true|false, "message": "...", "data": ... }
  • Fitur lanjutan (reporting, reminder, upload media internal, blacklist global, agent, dll) belum tersedia di versi ini dan akan membalas 501 Not Implemented.
  • Media lokal tersimpan di ./media; akses via http://host:port/media/<nama-file>.

Checklist implementasi

  1. Jalankan wa-gateway, set KEY jika ingin master key.
  2. Buat device via /api/device/create, simpan token.
  3. Scan QR via /api/device/scan hingga status online.
  4. Pasang webhook (opsional) untuk event masuk.
  5. Kirim pesan: pilih V1 (form) atau V2 (JSON) sesuai kebutuhan.
  6. Gunakan /api/device/info dan /api/v2/* untuk monitoring, penjadwalan, dan otomatisasi.
